# ยืนยันจำนวนรูปปิดงาน (Before / After) — ลูกค้า Pepsi

**อัปเดต:** 2026-05-22  
**สถานะ:** ยืนยันแล้ว — **ไม่จำกัดจำนวนรูป** (ตัวเลือก C)  
**อ้างอิง:** [MEETING-MINUTES.md](MEETING-MINUTES.md) ครั้งที่ 1 §6 · [WORK-PHASES.md](../WORK-PHASES.md) Phase 6

## สรุปที่ตกลงกับลูกค้า

| หัวข้อ | ค่าที่ใช้ |
|--------|-----------|
| จำนวนสูงสุดต่อ WO ต่อฝั่ง (Before / After) | **ไม่จำกัด** |
| แนะนำใน UI | ~**5 รูปต่อฝั่ง** (ไม่บังคับ) |
| ขนาดไฟล์ต่อรูป (อัปโหลด) | **ไม่จำกัด** — รับไฟล์จากกล้องมือถือรุ่นใหม่ · ย่อเป็น WebP (กว้างสูงสุด 1600px) ก่อนเก็บ DB |
| รูปแบบ | รับ `image/*` · เก็บ WebP |

เทียบ PHP เดิม: ไม่จำกัดจำนวนแถว · จำกัด **5 MB/ไฟล์** — ระบบใหม่ไม่จำกัดขนาดอัปโหลด (ไม่ใช้ `app.upload_max_mb` สำหรับ route นี้)

---

## นิยาม

- **1 ใบงาน (WO)** = หนึ่ง `idiw37`
- **ฝั่ง (phase)** = `ก่อนทำงาน (Before)` หรือ `หลังทำงาน (After)` — นับแยกกัน แต่**ไม่มีเพดานจำนวน**

## การ implement

| ส่วน | พฤติกรรม |
|------|----------|
| API `POST .../images` | ไม่ตรวจ `CONFIRM_IMAGE_LIMIT` |
| UI `ConfirmationImagesPanel` | ไม่ปิดการอัปโหลดเมื่อเกิน 5 หรือ 15 · แสดงข้อความแนะนำ ~5 |
| ค่าคงที่ | `CONFIRM_IMAGE_RECOMMENDED_PER_PHASE = 5` เท่านั้น (`confirm-image-phase.ts`, `confirm-image-limits.ts`) |
